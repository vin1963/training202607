# 題目一：商品管理系統（基礎 CRUD）— 學習說明文件

## 一、功能概述

商品管理系統實作了完整的 CRUD 操作，包含以下功能：

| 功能 | HTTP 請求 | 對應 DAO 方法 | SQL 操作 |
|---|---|---|---|
| 列表查詢 | GET /product/list | `findAll()` | SELECT (含軟刪除過濾) |
| 新增表單 | GET /product/add | — | — |
| 編輯表單 | GET /product/edit?id=X | `findById(id)` | SELECT WHERE id = ? |
| 刪除 | GET /product/delete?id=X | `delete(id)` | UPDATE SET deleted = 1 |
| 新增提交 | POST /product | `insert(product)` | INSERT + RETURN_GENERATED_KEYS |
| 編輯提交 | POST /product (含 id) | `update(product)` | UPDATE SET ... WHERE id = ? |
| 搜尋 | GET /product/search | `searchByName` / `findByCategory` | SELECT ... LIKE / WHERE category = ? |

```

    ▼
```

## 三、Model 層：Product.java (JavaBean)

```java
public class Product {
    private int id;              // 商品編號 (自動生成)
    private String name;         // 商品名稱
    private BigDecimal price;    // 商品價格 (精確十進位)
    private int stock;           // 庫存數量
    private String category;     // 商品類別
}
```

**重點說明：**
- `price` 使用 `BigDecimal` 而非 `double` / `float`，避免浮點數精度誤差
- 屬性命名與資料表欄位一致，便於 `mapRow()` 對映
- 提供無參數建構子（JSP EL 存取需要）與有參數建構子（方便建立）

## 四、DAO 層

### 4.1 ProductDao 介面（Contract）

```java
public interface ProductDao {
    void insert(Product product);                // C
    Optional<Product> findById(int id);          // R (單筆)
    List<Product> findAll();                     // R (多筆)
    List<Product> searchByName(String name);     // R (模糊)
    List<Product> findByCategory(String category); // R (類別)
    void update(Product product);                // U
    void delete(int id);                         // D (軟刪除)
    long count();                                // 總筆數
}
```

**設計原則：**
- 先定義介面再實作，Controller 只依賴 `ProductDao`，不依賴實作類別
- `findById` 回傳 `Optional<Product>` 明確表達可能查無資料
- `count()` 提供給分頁使用

### 4.2 ProductDaoImpl 實作詳解

#### CREATE — INSERT

```java
String sql = "INSERT INTO product (name, price, stock, category) VALUES (?, ?, ?, ?)";
try (Connection conn = DBUtil.getConnection();
     PreparedStatement ps = conn.prepareStatement(sql, Statement.RETURN_GENERATED_KEYS)) {
    ps.setString(1, product.getName());
    ps.setDouble(2, product.getPrice());
    ps.setInt(3, product.getStock());
    ps.setString(4, product.getCategory());
    ps.executeUpdate();
    try (ResultSet rs = ps.getGeneratedKeys()) {
        if (rs.next()) {
            product.setId(rs.getInt(1));   // 將自增 ID 回寫到物件
        }
    }
}
```

| 程式碼 | 說明 |
|---|---|
| `RETURN_GENERATED_KEYS` | 告訴 MySQL 回傳自動生成的 ID |
| `ps.setString(1, ...)` | 設定參數，從 1 開始，對應 `?` 順序 |
| `executeUpdate()` | 執行 INSERT/UPDATE/DELETE |
| `getGeneratedKeys()` | 取得自動產生的 ID 並寫回 JavaBean |

#### READ — SELECT (單筆)

```java
String sql = "SELECT * FROM product WHERE id = ? AND deleted = 0";
// ...
if (rs.next()) {
    return Optional.of(mapRow(rs));
}
return Optional.empty();
```

| 程式碼 | 說明 |
|---|---|
| `AND deleted = 0` | 軟刪除過濾，只回傳未刪除的資料 |
| `Optional.of()` / `Optional.empty()` | 明確表達查詢結果存在或不存在 |

#### READ — SELECT (多筆)

```java
String sql = "SELECT * FROM product WHERE deleted = 0";
List<Product> list = new ArrayList<>();
// ...
while (rs.next()) {
    list.add(mapRow(rs));
}
return list;
```

#### READ — 模糊搜尋

```java
String sql = "SELECT * FROM product WHERE deleted = 0 AND name LIKE ?";
ps.setString(1, "%" + name + "%");
```

| 程式碼 | 說明 |
|---|---|
| `LIKE ?` | 使用佔位符而非直接拼接，防止 SQL Injection |
| `"%" + name + "%"` | MySQL 端進行模糊比對 |

#### READ — 依類別查詢

```java
String sql = "SELECT * FROM product WHERE deleted = 0 AND category = ?";
ps.setString(1, category);
```

#### UPDATE

```java
String sql = "UPDATE product SET name = ?, price = ?, stock = ?, category = ? WHERE id = ? AND deleted = 0";
ps.setString(1, product.getName());
// ...
ps.setInt(5, product.getId());
ps.executeUpdate();
```

| 程式碼 | 說明 |
|---|---|
| `AND deleted = 0` | 避免更新已刪除的資料 |
| `ps.executeUpdate()` | 回傳受影響的列數，可判斷是否更新成功 |

#### DELETE — 軟刪除

```java
String sql = "UPDATE product SET deleted = 1 WHERE id = ?";
ps.setInt(1, id);
ps.executeUpdate();
```

**為什麼用軟刪除而不是 DELETE？**

| | 軟刪除 | 物理刪除 |
|---|---|---|
| SQL | `UPDATE SET deleted = 1` | `DELETE FROM ...` |
| 資料保留 | 保留 | 永久消失 |
| 可復原 | 可 | 不可 |
| 查詢 | 需加 `WHERE deleted = 0` | 不需過濾 |
| 適合場景 | 多數商業系統 | 暫存或測試資料 |

#### mapRow — ResultSet 轉 JavaBean

```java
private Product mapRow(ResultSet rs) throws SQLException {
    Product product = new Product();
    product.setId(rs.getInt("id"));
    product.setName(rs.getString("name"));
    product.setPrice(rs.getDouble("price"));
    product.setStock(rs.getInt("stock"));
    product.setCategory(rs.getString("category"));
    return product;
}
```

**每個 DAO 實作都應有 `mapRow` 私有方法**，職責單一：只負責將一筆 ResultSet 轉為 JavaBean。重複使用於 `findById`、`findAll`、`searchByName`、`findByCategory`。

## 五、Controller 層：ProductServlet.java

### 5.1 路由設計

```java
@WebServlet("/product/*")
public class ProductServlet extends HttpServlet {
    private ProductDao productDao = new ProductDaoImpl();
    // ...
}
```

`@WebServlet("/product/*")` 表示此 Servlet 處理所有以 `/product/` 開頭的請求。

### 5.2 doGet() — 處理所有 GET 請求

```
GET /product/list     → 查詢所有商品 → 轉發 productList.jsp
GET /product/add      → 直接轉發表單 (新增模式)
GET /product/edit?id=X → 查詢單筆 → 轉發表單 (編輯模式)
GET /product/delete?id=X → 軟刪除 → 重導向列表
GET /product/search   → 依 name 或 category 搜尋 → 轉發列表
```

**關鍵程式碼解析：**

```java
// 判斷路徑
String path = req.getPathInfo();  // 取得 /product 之後的路徑，如 /list, /edit

// 路徑分發 (if-else 結構)
if (path == null || path.equals("/") || path.equals("/list")) {
    handleList(req, resp);
} else if (path.equals("/edit")) {
    // ...
}
```

### 5.3 doPost() — 處理所有 POST 請求

```java
req.setCharacterEncoding("UTF-8");  // 防止中文亂碼

String idParam = req.getParameter("id");
// ...

// 資料驗證 (後端)
if (name.isEmpty()) {
    req.setAttribute("error", "名稱不可為空");
    req.getRequestDispatcher("/WEB-INF/views/productForm.jsp").forward(req, resp);
    return;  // 驗證失敗，返回表單
}

// 根據有無 id 決定新增或更新
if (idParam == null || idParam.isEmpty()) {
    productDao.insert(new Product(name, price, stock, category));
} else {
    Product product = new Product(name, price, stock, category);
    product.setId(Integer.parseInt(idParam));
    productDao.update(product);
}

resp.sendRedirect(req.getContextPath() + "/product/list");  // 完成後重導向列表
```

**PRG 模式 (Post-Redirect-Get)：** 新增/更新完成後使用 `sendRedirect` 而非 `forward`，避免使用者重新整理頁面時重複提交表單。

### 5.4 三層資料驗證

| 層級 | 位置 | 驗證內容 | 做法 |
|---|---|---|---|
| 前端 | productForm.jsp | 非空、格式 | `required` 屬性、`type="number"` |
| 後端 | ProductServlet.java | 名稱非空、價格 > 0、庫存 ≥ 0 | 條件判斷，不符則返回表單並顯示錯誤 |
| 資料庫 | MySQL | 非空、唯一 | 欄位 `NOT NULL`、`UNIQUE` |

## 六、View 層：JSP (EL + JSTL)

### 6.1 productList.jsp — 商品列表

```jsp
<%@ taglib uri="jakarta.tags.core" prefix="c" %>
<%@ taglib uri="jakarta.tags.fmt" prefix="fmt" %>
```

**搜尋表單：**
```jsp
<form action="${pageContext.request.contextPath}/product/search" method="get">
    名稱: <input type="text" name="name" />
    類別: <input type="text" name="category" />
    <button type="submit">搜尋</button>
</form>
```

**EL + JSTL 對照表：**

| 情境 | JSP 寫法 | 說明 |
|---|---|---|
| 空列表 | `<c:if test="${empty products}">` | 無資料時顯示提示 |
| 迴圈 | `<c:forEach var="p" items="${products}">` | 遍歷商品列表 |
| 貨幣格式 | `<fmt:formatNumber value="${p.price}" type="currency" currencySymbol="NT$" />` | 格式化貨幣 |
| 條件樣式 | `<c:if test="${p.stock < 10}"><span style="color:red;">` | 庫存不足紅字 |
| XSS 防護 | `<c:out value="${p.name}" />` | 跳脫 HTML 標籤 |
| URL 路徑 | `${pageContext.request.contextPath}/product/edit?id=${p.id}` | 產生正確路徑 |

**庫存不足紅字邏輯：**
```jsp
<c:if test="${p.stock < 10}">
    <span style="color:red;"><c:out value="${p.stock}" /></span>
</c:if>
<c:if test="${p.stock >= 10}">
    <c:out value="${p.stock}" />
</c:if>
```

**刪除確認：**
```jsp
<a href="..." onclick="return confirm('確定要刪除此商品嗎？');">刪除</a>
```
點擊時彈出確認對話框，按取消則不執行連結。

### 6.2 productForm.jsp — 新增/編輯表單

**判斷模式（新增 vs 編輯）：**
```jsp
<c:choose>
    <c:when test="${not empty product}">編輯商品</c:when>
    <c:otherwise>新增商品</c:otherwise>
</c:choose>
```

**共用表單技巧：**
```jsp
<form action="${pageContext.request.contextPath}/product" method="post">
    <%-- 編輯時才帶入 id，Servlet 據此判斷 CREATE 或 UPDATE --%>
    <c:if test="${not empty product}">
        <input type="hidden" name="id" value="<c:out value='${product.id}' />" />
    </c:if>

    <%-- 表單欄位 value 在有 product 時帶入資料，否則為空 --%>
    <input type="text" name="name" value="<c:out value='${product.name}' />" />
    <input type="number" name="price" step="0.01" value="<c:out value='${product.price}' />" />
    <input type="number" name="stock" value="<c:out value='${product.stock}' />" />
</form>
```

**錯誤訊息顯示：**
```jsp
<c:if test="${error != null}">
    <p style="color:red;"><c:out value="${error}" /></p>
</c:if>
```

## 七、請求流程圖 (完整 CRUD)

### CREATE：新增商品

```
瀏覽器 GET /product/add
  → ProductServlet.doGet()
    → forward → productForm.jsp (空表單，無 product)

使用者填寫表單後送出
  → 瀏覽器 POST /product (name=iPhone&price=29900&stock=50&category=手機)
    → ProductServlet.doPost()
      → 資料驗證
      → productDao.insert(new Product(...))
        → INSERT INTO product ... RETURN_GENERATED_KEYS
        → 取得自增 ID 寫回 Product 物件
      → resp.sendRedirect(/product/list)
```

### READ：列表查詢

```
瀏覽器 GET /product/list
  → ProductServlet.doGet()
    → productDao.findAll()
      → SELECT * FROM product WHERE deleted = 0
    → req.setAttribute("products", list)
    → forward → productList.jsp
      → <c:forEach> 遍歷 products 產生表格
```

### UPDATE：編輯商品

```
瀏覽器 GET /product/edit?id=1
  → ProductServlet.doGet()
    → productDao.findById(1)
      → SELECT * FROM product WHERE id = 1 AND deleted = 0
      → Optional<Product>
    → req.setAttribute("product", product)
    → forward → productForm.jsp (表單帶入資料)

使用者修改後送出
  → 瀏覽器 POST /product (id=1&name=iPhone15&price=27900&stock=45&category=手機)
    → ProductServlet.doPost()
      → idParam 不為空 → UPDATE 模式
      → productDao.update(product)
        → UPDATE product SET name=?, price=?, stock=?, category=? WHERE id=? AND deleted=0
      → resp.sendRedirect(/product/list)
```

### DELETE：刪除商品

```
瀏覽器 GET /product/delete?id=1
  → ProductServlet.doGet()
    → productDao.delete(1)
      → UPDATE product SET deleted = 1 WHERE id = 1  ← 軟刪除
    → resp.sendRedirect(/product/list)
```

### SEARCH：搜尋商品

```
瀏覽器 GET /product/search?name=iPhone
  → ProductServlet.doGet()
    → name 不為空 → productDao.searchByName("iPhone")
      → SELECT * FROM product WHERE deleted = 0 AND name LIKE '%iPhone%'
    → req.setAttribute("products", list)
    → forward → productList.jsp
```

## 八、DBUtil.java — 資料庫連線工具

```java
public class DBUtil {
    private static final String URL = "jdbc:mysql://localhost:3306/practice_db"
            + "?useSSL=false&serverTimezone=Asia/Taipei&characterEncoding=utf8";
    private static final String USER = "root";
    private static final String PASSWORD = "password";

    static {
        try {
            Class.forName("com.mysql.cj.jdbc.Driver");
        } catch (ClassNotFoundException e) {
            throw new RuntimeException("MySQL Driver not found", e);
        }
    }

    public static Connection getConnection() throws SQLException {
        return DriverManager.getConnection(URL, USER, PASSWORD);
    }
}
```

| 設定 | 說明 |
|---|---|
| `useSSL=false` | 關閉 SSL（開發環境） |
| `serverTimezone=Asia/Taipei` | 設定時區 |
| `characterEncoding=utf8` | 設定字元編碼 |
| `Class.forName(...)` | 載入 JDBC Driver (MySQL 8 以上可省略) |

## 十、EncodingFilter.java — 編碼過濾器

```java
@WebFilter("/*")
public class EncodingFilter implements Filter {
    public void doFilter(ServletRequest request, ServletResponse response,
                         FilterChain chain) throws IOException, ServletException {
        request.setCharacterEncoding("UTF-8");
        response.setCharacterEncoding("UTF-8");
        chain.doFilter(request, response);
    }
}
```

**為什麼需要 Filter？** 若不設定編碼，`req.getParameter("name")` 取得到的中文會是亂碼。`@WebFilter("/*")` 表示所有請求都經過此 Filter。

## 十一、軟刪除 (Soft Delete) 模式

### 資料表結構

```sql
CREATE TABLE product (
    id      INT AUTO_INCREMENT PRIMARY KEY,
    name    VARCHAR(100) NOT NULL,
    price   DECIMAL(10,2) NOT NULL,
    stock   INT NOT NULL DEFAULT 0,
    category VARCHAR(50),
    deleted TINYINT DEFAULT 0     -- 0: 正常, 1: 已刪除
);
```

### 對 DAO 的影響

| 操作 | 一般寫法 | 軟刪除寫法 |
|---|---|---|
| 新增 | 不變 | 不變（deleted 預設 0） |
| 查詢 | `SELECT * FROM product` | `SELECT * FROM product WHERE deleted = 0` |
| 修改 | `UPDATE ... WHERE id = ?` | `UPDATE ... WHERE id = ? AND deleted = 0` |
| 刪除 | `DELETE FROM product WHERE id = ?` | `UPDATE product SET deleted = 1 WHERE id = ?` |

---

## 十二、PreparedStatement 防 SQL Injection

### ❌ 錯誤寫法（字串拼接）

```java
String sql = "SELECT * FROM product WHERE name LIKE '%" + name + "%'";
```

使用者輸入 `' OR 1=1 --` 時，SQL 變成：
```sql
SELECT * FROM product WHERE name LIKE '%' OR 1=1 --%'
```
**所有資料都被查出，安全性漏洞。**

### ✅ 正確寫法（使用 ? 佔位符）

```java
String sql = "SELECT * FROM product WHERE name LIKE ?";
ps.setString(1, "%" + name + "%");
```

`PreparedStatement` 會自動跳脫特殊字元，`?` 的位置只會被當作「值」而非「SQL 語法」。

## 十三、try-with-resources 資源管理

```java
try (Connection conn = DBUtil.getConnection();
     PreparedStatement ps = conn.prepareStatement(sql);
     ResultSet rs = ps.executeQuery()) {
    // 使用 conn, ps, rs
} catch (SQLException e) {
    throw new RuntimeException(e);
}
// 離開 try 區塊時，rs, ps, conn 會自動關閉 (AutoCloseable)
```

**對比舊寫法：** 以前需要在 `finally` 區塊中逐一 `close()`，容易漏掉導致連線泄漏。

## 十五、常見錯誤排解

| 錯誤現象 | 原因 | 解決方式 |
|---|---|---|
| `ClassNotFoundException: com.mysql.cj.jdbc.Driver` | MySQL Connector 未加入 | 確認 pom.xml 有 `mysql-connector-j` |
| 500 JasperException | JSTL jar 缺失 | 確認 JSTL API + Implementation 依賴 |
| 404 | URL 路徑錯誤 | 確認 `@WebServlet("/product/*")` 與 JSP 中 `contextPath` 一致 |
| 中文變成 `???` | 編碼未設定 | 確認 EncodingFilter 存在且正確 |
| `Table 'practice_db.product' doesn't exist` | 資料表未建立 | 執行 SQL 建表 |
| `Data too long for column 'name'` | 資料長度超過欄位限制 | 調整 VARCHAR 長度或檢查輸入 |
| `Duplicate entry for key` | 違反 UNIQUE 約束 | 檢查資料是否重複 |
